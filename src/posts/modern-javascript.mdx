## Modern Javascript

- Use TypeScript
- If you can, use [Bun](https://bun.sh). It's faster than Node, has a much better DX, great built-in APIs including tests, and it doesn't impose any config overhead to using TypeScript
  - You may not be able to use Bun, for example if you're using Next.js, React Native, or some other incompatible framework or library
  - If you can't use Bun, use the latest version of Node
- For websites, apps, and APIs, use [Next.js 13.4 with the App router](https://nextjs.org/docs)
- Fetch data and make other HTTP requests using the `fetch` API. Don't install a dependency for HTTP fetching
- Validate your data at serialization/deserialization boundaries with [Zod](https://zod.dev/)
- If you don't need replication, use [SQLite](https://sqlite.org/)
  - Don't use SQLite if you _do_ need replication
- If you need replication, use MySQL or Postgres.
  - [PlanetScale](https://planetscale.com/) has a great MySQL offering
  - [Neon](https://neon.tech/) or [Vercel Postgres](https://vercel.com/docs/storage/vercel-postgres) are great offerings if your database isn't hundreds of GB or more
- Don't use an ORM. Set up your project to work well with SQL, and define your schemas in SQL DDL
  - Treat SQL with respect. It belongs in your codebase
